## Design Context

### Users
The primary users are Django developers seeking high-performance, hypermedia-driven interactivity without JavaScript fatigue. They value technical transparency, efficiency, and a "code-first" philosophy.

### Brand Personality
**Technical, Minimal, Brutalist.**
The voice is direct and authoritative, evoking the "technical confidence" of a well-configured terminal environment.

### Aesthetic Direction
**Light Brutalist / TUI-inspired.** 
The interface is strictly light mode, utilizing a palette of white, gray, and black. It mimics Terminal User Interfaces (TUI) with sharp 0px corners, prominent borders, and dense, structured layouts.

### Design Principles
1. **TUI-First Architecture**: Layouts should use borders, logical blocks, and monospaced grids to mimic the structure of terminal applications.
2. **Zero-Smoothing Brutalism**: Strictly no rounded corners (`radius: 0px`), no gradients, and no soft shadows. Visual hierarchy is achieved through border weight and high-contrast typography.
3. **Monospaced Hierarchy**: Use `IBM Plex Mono` for all functional text and `Doto` for high-impact headers to reinforce the terminal aesthetic.
4. **Functional Density**: Prioritize information density and scanability over decorative whitespace. Every element should feel like a necessary part of a technical tool.
